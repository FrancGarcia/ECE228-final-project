# Object, Depth, and Trajectory Estimation for Autonomous Vehicles

ECE 228 - Scene Reconstruction for Autonomous Driving
Team 56: Aidan Manternach, Manasvin Surya BJ, Francisco Garcia, Harini Sabapathy

This project builds a camera-only 3D scene reconstruction pipeline for autonomous driving. From a single monocular camera stream, it detects road agents, estimates their metric depth, projects them into a bird's-eye-view (BEV), tracks them across frames, and predicts their trajectories. The idea is inspired by the Tesla Full Self-Driving visualization and extended to explicit trajectory prediction.

## What this project does

Autonomous vehicles need to understand and predict how their environment is moving, and they have to do it in real time under limited compute and memory. Our goal is an end-to-end pipeline that reconstructs the 3D scene and forecasts agent motion using only camera input, with no LiDAR at inference time.

The pipeline has four stages:

| Stage | Method | Output |
|---|---|---|
| 1. Object Detection | YOLOv8m fine-tuned on KITTI | 2D bounding boxes for Car, Pedestrian, Cyclist |
| 2. Depth Estimation | Depth Anything V2 (Metric, Outdoor) | Per-pixel metric depth in metres |
| 3. BEV Fusion | Camera intrinsics unprojection | Top-down bird's-eye-view of all agents |
| 4. Trajectory Prediction | Kalman filter + ego-motion compensation | World-frame tracks, velocities, crossing intent |

We use the KITTI dataset for training and evaluation. KITTI provides real-world driving sequences from a moving vehicle with ground-truth object labels and LiDAR depth.

Pipeline flow:

1. Run YOLOv8m on the camera frame to get bounding boxes.
2. Run Depth Anything V2 on the same frame to get a dense metric depth map.
3. Sample the depth under each box and unproject it to 3D using the camera intrinsics.
4. Place the 3D points on a top-down BEV canvas.
5. Track agents across frames with a Kalman filter and remove ego-vehicle motion.
6. Predict each agent's short-term trajectory and crossing intent.

## Repository structure

| Folder | Contents |

The research and development is in the notebooks (`yolov8-finetuning`, `depth-model`, `bev_fusion`, `bev_trajectory_modelling`). The `bev_inference/` folder packages the pipeline into command-line scripts that reproduce the results on any machine.

## Quick start

All runnable code lives in [`bev_inference/`](bev_inference/). It auto-detects the best device (`cuda`, then `mps`, then `cpu`) and runs on Colab, macOS (Apple Silicon), Linux/Windows GPUs, and CPU-only machines.

### 1. Install dependencies

```bash
cd bev_inference
pip install -r requirements.txt
```

On Google Colab, append `--install-deps` to any script instead of running the install step separately.

### 2. Run on a sequential KITTI drive

```bash
# Download 50 consecutive frames from a single KITTI raw drive (~800 MB)
python download_kitti.py

# Process all frames into per-frame PNGs, GIFs, and validation stats
python run_kitti.py
```

### 3. Run on your own images

```bash
# Drop .png/.jpg files into bev_inference/inputs/, then:
python run_inputs.py
```

### 4. Run on a video

```bash
# Drop a landscape video into bev_inference/vid_input/, then:
python run_video.py
```

The video script extracts temporally ordered frames (sampling at a target FPS and down-scaling to a sensible processing width), then runs the same detection, depth, and BEV pipeline.

### Outputs

Every run rewrites `bev_inference/outputs/` with:

- Per-frame PNGs: RGB with detections, depth map, BEV canvas, and a 3-panel composite
- Animated GIFs: `detection.gif`, `depth.gif`, `bev.gif`, `composite.gif`
- `summary.json`: per-frame detection counts, timings, and 3D coordinates
- A terminal validation report: detections per frame, FPS, per-class depth and confidence, and depth distribution

See [`bev_inference/README.md`](bev_inference/README.md) for the full CLI reference.

## Models

- YOLOv8m fine-tuned on KITTI (Car, Pedestrian, Cyclist). The trained checkpoint is at [`yolov8-finetuning/best.pt`](yolov8-finetuning/best.pt) and is the default weights for every inference script.
- Depth Anything V2 (`depth-anything/Depth-Anything-V2-Metric-Outdoor-Small-hf`), downloaded automatically from HuggingFace on first run.

## Reproducing the research

The notebooks are written for Google Colab with a GPU (T4 or better). Each one lists its execution order in the first cell:

1. `yolov8-finetuning/yolov8_kitti_finetune.ipynb`: fine-tune YOLOv8m, evaluate mAP, export `best.pt`
2. `depth-model/depth_model.ipynb`: evaluate Depth Anything V2 against KITTI LiDAR
3. `bev_fusion/bev_fusion_1.ipynb`: fuse detection and depth into BEV
4. `bev_trajectory_modelling/bev_trajectory_prediction_with_validation.ipynb`: full tracking and trajectory prediction with validation

For day-to-day reproduction of results, use the scripts in `bev_inference/`.
